# CrackGate Go-Live Runbook (AWS)

End-to-end ~90 min. Re-run any step idempotently; nothing here is one-shot.

> **Target stack:** Mumbai (`ap-south-1`), EC2 `t4g.small` + RDS `db.t4g.micro`,
> Cloudflare DNS (no CloudFront yet), GitHub Actions CI/CD via OIDC.
> See [docs/architecture.md](docs/architecture.md) for the picture.

## 0 · Prereqs (one-time per AWS account)

```bash
# 1. AWS CLI configured + admin/poweruser
aws sts get-caller-identity

# 2. Terraform 1.6+
brew install terraform   # or asdf install terraform 1.9.0

# 3. Generate an SSH keypair for the EC2 deploy user (or reuse an existing one)
ssh-keygen -t ed25519 -C "crackgate-deploy" -f ~/.ssh/crackgate-deploy
# pubkey goes into terraform.tfvars below; private key into GitHub Actions secret

# 4. Bootstrap the Terraform state backend (one-time)
aws s3api create-bucket --bucket crackgate-tfstate --region ap-south-1 \
  --create-bucket-configuration LocationConstraint=ap-south-1
aws s3api put-bucket-versioning --bucket crackgate-tfstate \
  --versioning-configuration Status=Enabled
aws dynamodb create-table --table-name crackgate-tflock \
  --attribute-definitions AttributeName=LockID,AttributeType=S \
  --key-schema AttributeName=LockID,KeyType=HASH \
  --billing-mode PAY_PER_REQUEST --region ap-south-1

# 5. Create the GitHub OIDC provider (one-time per account)
aws iam create-open-id-connect-provider \
  --url https://token.actions.githubusercontent.com \
  --client-id-list sts.amazonaws.com \
  --thumbprint-list 6938fd4d98bab03faadb97b34396831e3780aea1
```

## 1 · Provision AWS infra

```bash
cd infra-tf
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars:
#   admin_ip_cidr      = "$(curl -s ifconfig.me)/32"
#   deploy_ssh_pubkey  = "$(cat ~/.ssh/crackgate-deploy.pub)"

terraform init
terraform plan -out tfplan
terraform apply tfplan
```

Apply prints a `next_steps` block — keep that terminal open.

## 2 · Build `.env.production`

Grab the autogenerated RDS password ONCE:

```bash
cd infra-tf
terraform output -raw rds_master_password
terraform output -raw rds_endpoint
terraform output -raw ecr_registry
```

Then in repo root:

```bash
cp .env.production.aws.example .env.production
```

Fill in:

| Var | Source |
|---|---|
| `RDS_HOST` | `terraform output rds_endpoint` |
| `POSTGRES_PASSWORD` | `terraform output -raw rds_master_password` |
| `ECR_REGISTRY` | `terraform output ecr_registry` |
| `ECR_REPO` | `terraform output -raw ecr_repo_uri` |
| `IMAGE_TAG` | `latest` (CI overwrites on every deploy) |
| `BACKUP_BUCKET` | `crackgate-prod-backups` |
| `UPLOADS_BUCKET` | `crackgate-prod-uploads` |
| `AUTH_SECRET` | `openssl rand -base64 48` |
| `AUTH_URL` | `https://crackgate.in` |
| `GOOGLE_CLIENT_ID` / `_SECRET` | Google Cloud Console (see §6) |
| `RAZORPAY_*` | LIVE keys when ready; for now use `rzp_test_*` (payments work in test mode) |
| `ACME_EMAIL` | your email — Let's Encrypt cert recovery |
| `SENTRY_DSN` | leave blank if not set up |

## 3 · Upload secret

```bash
aws secretsmanager put-secret-value \
  --region ap-south-1 \
  --secret-id crackgate/prod/env \
  --secret-string file://.env.production

# verify
aws secretsmanager get-secret-value --region ap-south-1 \
  --secret-id crackgate/prod/env --query SecretString --output text | head -5
```

**Delete the local `.env.production` immediately after.** It's git-ignored but
shouldn't sit on disk longer than this step.

## 4 · Wait for EC2 bootstrap

```bash
EIP=$(cd infra-tf && terraform output -raw ec2_public_ip)
ssh -i ~/.ssh/crackgate-deploy ubuntu@$EIP

# on the box
sudo cloud-init status --wait
sudo tail -100 /var/log/cloud-init-output.log
# you should see: "✓ EC2 bootstrap complete."

ls -la /opt/crackgate/.env.production    # bootstrap pulled it from Secrets Manager
systemctl status ecr-login.timer crackgate-backup.timer
docker info | head
```

## 5 · Cloudflare DNS

In the Cloudflare dashboard for `crackgate.in`:

| Type | Name | Content | Proxy |
|---|---|---|---|
| A | `@` | `<EIP>` | **DNS-only (grey cloud)** initially |
| A | `www` | `<EIP>` | DNS-only |

The proxy MUST be off the first time so Caddy can pass Let's Encrypt's HTTP-01
challenge. After the cert issues (step 7), flip to "Proxied (orange cloud)"
for free DDoS protection.

Wait 2–5 min for propagation:
```bash
dig +short crackgate.in @1.1.1.1   # should return the EIP
```

## 6 · Google OAuth production client

1. https://console.cloud.google.com/apis/credentials
2. Create OAuth 2.0 Client ID → Web application
3. Authorized JavaScript origins: `https://crackgate.in`
4. Authorized redirect URIs: `https://crackgate.in/api/auth/callback/google`
5. Copy Client ID + Secret into `.env.production` and re-upload to Secrets Manager

## 7 · GitHub Actions setup

In the repo Settings → Secrets and variables → Actions:

**Repository secrets:**
```
AWS_DEPLOY_ROLE_ARN  = (terraform output gha_deploy_role_arn)
ECR_REGISTRY         = (terraform output ecr_registry)
EC2_HOST             = (terraform output ec2_public_ip)
EC2_SSH_KEY          = (cat ~/.ssh/crackgate-deploy)
SLACK_WEBHOOK_URL    = (optional; failure pings)
NEXT_PUBLIC_POSTHOG_KEY = (optional)
```

**Repository variables:**
```
NEXT_PUBLIC_GOOGLE_ENABLED         = 1
NEXT_PUBLIC_WHATSAPP_ENABLED       = 0
NEXT_PUBLIC_WHATSAPP_NUMBER        = ...
NEXT_PUBLIC_WHATSAPP_COMMUNITY_URL = ...
NEXT_PUBLIC_RAZORPAY_KEY_ID        = rzp_live_... or rzp_test_...
NEXT_PUBLIC_UPI_VPA                = ...
NEXT_PUBLIC_UPI_PAYEE_NAME         = CrackGate
NEXT_PUBLIC_POSTHOG_HOST           = https://app.posthog.com
```

Create a `production` environment (Settings → Environments → New environment)
so the `deploy` job's `environment: production` directive resolves.

## 8 · First deploy

```bash
git add -A
git commit -m "chore: trigger first prod deploy"
git push origin main
```

Watch in the Actions tab. Steps:
1. `verify` — lint + typecheck (~2 min)
2. `build` — ARM64 docker build + push to ECR (~5 min cold cache)
3. `deploy` — SSH to EC2, pull, migrate, restart web, healthcheck (~2 min)

If `deploy` fails the rollback is automatic — the workflow re-pins `IMAGE_TAG=latest`
and restarts `web`.

## 9 · Verify

```bash
curl -fsS https://crackgate.in/api/healthz   # → {"ok":true}
curl -fsI https://crackgate.in | head -5    # → 200 OK + HSTS
open https://crackgate.in
```

Check on the EC2 box:
```bash
ssh -i ~/.ssh/crackgate-deploy deploy@$EIP
cd /opt/crackgate
docker compose -f docker-compose.aws.yml ps      # 3 running: pgbouncer, web, caddy
docker compose -f docker-compose.aws.yml logs --tail=50 web
```

Flip Cloudflare proxy to Proxied (orange cloud) now that the cert is valid.

## 10 · Razorpay webhook

Dashboard → Settings → Webhooks → Add webhook:
- URL: `https://crackgate.in/api/payments/webhook`
- Events: `payment.captured`, `payment.failed`, `subscription.activated`
- Secret: same value as `RAZORPAY_WEBHOOK_SECRET` in `.env.production`

## 11 · Trigger first nightly backup manually (sanity check)

```bash
ssh -i ~/.ssh/crackgate-deploy deploy@$EIP
cd /opt/crackgate
docker compose -f docker-compose.aws.yml --env-file .env.production \
  --profile backup run --rm pg-backup

# Verify in S3
aws s3 ls s3://crackgate-prod-backups/db/
```

Then trigger the systemd timer-based run on schedule:
```bash
systemctl list-timers crackgate-backup.timer
```

## 12 · Restore drill (do this once!)


## Day-2 ops

### Push a code change
```bash
git push origin main   # GitHub Actions does the rest
```

### View live logs
```bash
aws logs tail /crackgate/web --follow --region ap-south-1
```

### Rotate a secret (e.g. AUTH_SECRET)
```bash
# Pull, edit, re-upload
aws secretsmanager get-secret-value --secret-id crackgate/prod/env \
  --query SecretString --output text > /tmp/env
$EDITOR /tmp/env
aws secretsmanager put-secret-value --secret-id crackgate/prod/env \
  --secret-string file:///tmp/env
shred -u /tmp/env

# Restart web on the EC2 box to pick up the new env
ssh -i ~/.ssh/crackgate-deploy deploy@$EIP \
  'sudo systemctl restart crackgate.service'
```

### Scale EC2 vertically (zero downtime not possible on single box)
```bash
# infra-tf/terraform.tfvars
ec2_instance_type = "t4g.medium"

terraform apply   # stops, resizes, starts — ~3 min outage
```

### Restore from backup
See §12.

### Tear down (CAUTION — deletion_protection = true so this fails by default)
```bash
# 1. Disable RDS deletion protection
aws rds modify-db-instance --db-instance-identifier crackgate-prod-pg \
  --no-deletion-protection --apply-immediately

# 2. Then
cd infra-tf
terraform destroy
```

```bash
  --alarm-name crackgate-billing \
  --metric-name EstimatedCharges --namespace AWS/Billing \
  --statistic Maximum --period 21600 --threshold 50 \
  --comparison-operator GreaterThanThreshold \
  --dimensions Name=Currency,Value=USD \
  --region us-east-1
```
